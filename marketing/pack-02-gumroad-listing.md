## Long description

**40 prompts organized by Cursor mode — Composer, Chat, Inline, Rules, and session flow. No generic "AI coding prompts" you can Google.**

Every prompt in this pack is labeled for the mode it's designed for. Composer prompts assume multi-file agent changes. Chat prompts are one-shots. Inline prompts work with Cmd-K. Rules go into your `.cursorrules` file and apply everywhere.

## What's inside

**5 sections, 40 prompts total:**

1. **Composer (10)** — multi-file refactors with explicit touch lists, library upgrades with breaking change mapping, monolith decomposition, CRUD generation from schema, deprecated-pattern retirement
2. **Chat (10)** — why-is-this-here archaeology, breakage prediction, tradeoffs grounded in your actual repo, "is there already a helper for this?", staff-engineer PR grading
3. **Inline (10)** — tighten without changing behavior, type without refactoring, extract only when reuse exists, docstrings that say something, structured logging that helps
4. **.cursorrules (5)** — paste-ready rule sets for TypeScript monorepo, Python FastAPI, conservative refactor mode, no-comments-unless-WHY, test philosophy
5. **Session flow (5)** — start with orientation, end with handoff notes, detect when you're stuck vs progressing, narrow a too-big task, recover from a broken Composer session
